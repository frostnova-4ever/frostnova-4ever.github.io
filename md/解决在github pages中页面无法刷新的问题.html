解决在github pages中页面无法刷新的问题

当静态网页部署在 github 的 pages 页面时，排除网络问题后，点击浏览器刷新按钮却不能正常显示？

　　我在部署油 vue 转静态 js 页面时也遇到相同问题，经研究，我认为是如下问题：

　　路由问题：GitHub Pages 上出现刷新后 404 错误，通常是因为使用了单页应用（SPA）路由，而 GitHub Pages 默认仅支持基于文件的路由。当刷新页面时，浏览器直接请求特定路径（如 yourdomain.com/page），但 GitHub Pages 会尝试查找名为 page 的实际文件，而不是将请求路由到 index.html。

如何解决？

　　方法 1.添加 404.html 页面

　　创建一个 404.html​ 文件，内容与 index.html​ 相同，或使用 JavaScript 将所有 404 错误重定向到 index.html​（这个方法比较逆天，是在 404 错误上修改定向）

　　方法 2.修改路由

　　 针对特定框架修改：

　　 vue：在 vue 中，如果上传 github pages，需要调整为 Hash 路由，(将 createWebHistory()​ 改为 createWebHashHistory()​)，这样所有路由都变成/#/page/2 形式，GitHub Pages 能正确处理。

import { createRouter, createWebHashHistory } from 'vue-router';
// ...existing code...
export default createRouter({
    history: createWebHashHistory(),
    routes
});

　　这样就应该能解决该问题了
